Core Features

六大核心能力

覆盖视障用户出行全场景，一部手机即可使用，零额外硬件成本。

👁️

实时场景识别

ML Kit Image Labeling 端侧推理，3秒内将摄像头画面转化为结构化描述。结合像素亮度与色调分析，输出自然语言场景播报。

⚠️

障碍物检测预警

实时识别行人、车辆、动物等高危目标，建立优先级队列，触发震动+语音双重预警。端侧处理，无需网络。

🧭

步行语音导航

高德 v3 步行 API + WGS84→GCJ02 坐标转换，语音输入目的地，路线逐段拆解为自然语言播报。

📄

OCR 文字朗读

ML Kit Text Recognition 端侧文字提取，覆盖药品说明书、快递面单、路牌等日常场景。

🎤

语音交互系统

原生 SpeechRecognizer 封装，支持无 Google 服务的国产设备。全功能语音控制，零视觉依赖。

🔊

TTS 双引擎容灾

系统TTS + 百度在线TTS自动降级 + 音频焦点管理。将国产设备语音可用率提升至接近 100%。

Architecture

技术架构

端侧感知 + 语音输出，将视觉世界翻译为听觉信息。

感知层 · 端侧 AI 三引擎并行
场景识别、障碍检测、OCR 共享摄像头数据流，帧调度避免资源竞争。全部端侧推理，无云端依赖，保护隐私。
交互层 · 全语音闭环
双引擎容灾 TTS + 原生语音识别。链路：语音指令 → 功能执行 → AI 分析 → 语音播报，全程无需视觉操作。
导航层 · 高德 API + 坐标纠偏
GPS WGS-84 实时转换 GCJ-02，消除 100-700m 偏移。路线逐段转自然语言，降低认知负荷。

Innovation

核心创新点

针对国产 Android 设备生态的三大技术突破。

1

TTS 端云容灾机制

去除临时对象竞态 + 延迟重试 + 在线降级 + AudioFocusRequest 音频焦点管理。一套代码兼容 ColorOS、MIUI 等主流国产系统，解决行业级语音合成适配难题。

2

端侧多模型复用架构

三大 AI 引擎共享摄像头数据流与计算资源，帧调度 + 优先级队列避免资源竞争。单设备运行全部模型，无需云端，离线可用。

3

全语音无障碍交互范式

从权限授权到功能操作全链路语音驱动。WGS84→GCJ02 坐标纠偏确保导航精度。可推广至驾驶辅助、智能家居等场景。
